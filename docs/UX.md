# BusyElf UI/UX 设计

> 术语对应 [PROTOCOL.md](PROTOCOL.md) 的状态:`working`(在干活,阻止休眠)、`waiting`(等用户,放行休眠但需关注)、`done`(已完成,终态,放行)、`failed`(异常停止,终态,放行)。

## 菜单栏图标

用单一 SF Symbol **`bolt.fill`(⚡︎)**,寓意"通电 / 保持清醒"。**只靠明暗、数字、着色、右上角合成角标传达状态,绝不替换字形**——换字形会改变宽度、让菜单栏抖动。`bolt.fill` 平时是 template 图,自动适配深浅色菜单栏;需要画角标时才按当前外观合成一张非 template 图(见 `UI/StatusItemController.swift`)。

`button.imagePosition = .imageLeading`,图标在左、数字在右。

| 状态 | 外观 | 含义 |
|---|---|---|
| 空闲(无活动任务、无角标) | `[ ⚡︎ ]` 半透明(alpha≈0.45),**无数字** | 允许休眠 |
| 有 working | `[ ⚡︎ 2 ]` 全亮 + 数字 | 正在阻止休眠 |
| 有 waiting | `[ ⚡︎ 2 ]` 烤橙图 + 橙数字 | 有 agent 需要你处理 |
| 有未看 **完成**(无活动) | `[ ⚡︎•]` **整只闪电烤绿** + 右上角**绿点** | 顶层任务刚完成,点开看一眼 |
| 有未看 **失败** | `[ ⚡︎•]` **整只闪电烤红** + 右上角**红点** | 有任务异常停止,紧急 |

- 数字 = **活动任务数**(working + waiting),钳制为 `9+`。**终态(done/failed)不计入数字**,只用角标表达,避免数字虚高。**"疑似已断"(working 但超过无活动阈值,看门狗已放行休眠)也不计入数字**——图标不再显忙,与"已放行休眠"一致(见 [DESIGN.md](DESIGN.md) §6)。
- **完成绿点只由顶层任务点亮**:子任务(subagent)完成**静默**——不点亮菜单栏绿点、不通知(它们数量多、是父任务的内部步骤)。父/顶层任务完成才亮绿。
- **完成的"整只烤绿"只在无活动任务时**(与失败"整只烤红"统一 UX):仍有 working/waiting 时只保留右上角绿点、底图仍随忙碌态(template/烤橙),避免"看着像全好了"误导休眠判断。失败红优先级最高,任何时候都整只烤红。
- 角标优先级:**红(未看失败) > 绿(未看完成)**;底图着色优先级:**失败红 > 等待橙 > 完成绿(仅无活动时)> 常规 template**。
- "只有终态项"时(活动数=0 但有角标),图标**正常亮显 + 角标**,而非半透明空闲态。
- 角标是"看一次就清"的提示:打开 popover 即标记 seen → 角标消失(见下"终态生命周期")。

```
空闲:            [ ⚡︎ ]            (半透明)
2 个在干活:       [ ⚡︎ 2 ]
1 干活 1 等你:    [ ⚡︎ 2 ]          (烤橙)
顶层任务刚完成:   [ ⚡︎•]            (无活动时整只烤绿 + 绿点;子任务完成静默)
有任务失败:       [ ⚡︎•]            (整只烤红 + 红点,优先)
```

交互:左键打开 popover;右键弹精简 `NSMenu`(打开面板、开关、**实际监听地址(只读)**、**「监听所有网口 (0.0.0.0)」复选**、About、Quit)。监听地址形如 `监听:127.0.0.1:17872`(开了所有网口则 `0.0.0.0:…`),方便对齐适配器 URL;切换网口即热重启监听(首次绑非回环会触发系统本地网络授权)。

## Popover

`NSPopover` 锚定状态栏按钮,固定宽 **340pt**,内容用**纯 AppKit**(`NSViewController` + `NSStackView`/`NSView`,**不用** `NSHostingController`、不链接 SwiftUI——这是 ~12MB footprint 的关键,见 DESIGN.md §2)。结构:状态头 → 可滚动任务列表(封顶 320pt,超出滚动)→ 细分隔线 → 底部设置(常用开关 +「更多设置」折叠区)。

> **尺寸**:列表高度 = `min(内容高, 320)`,popover 总高 = 逐项确定性求和(状态头 / 底部 / 分隔线各自 intrinsic + 内容区)。**不**用含 `NSScrollView` 的 `view.fittingSize`(对滚动视图层级不可靠,曾导致"撑大缩不回")。见 `PopoverController.syncContentSize` 与 [AGENTS.md](../AGENTS.md) 调试教训。

### 活跃 + 终态混排

```
+--------------------------------------------------+
|  ⚡︎  BusyElf                                  ⋯  |
|  Blocking sleep · 4 个任务                       |
+--------------------------------------------------+
|  ● api-server                                ×   |   ← 绿点 working
|    🔧 Bash: pytest -x …                          |   ← 当前动作(tool 优先,退化 reply)
|    12m 04s · claude-code · 修复登录超时…          |   ← 时长 · 来源 · 用户提示词(截断)
|  ----↳ 子任务紧跟父行,缩进、无分隔线------------- |
|    ↳ Explore                                 ×   |   ← 子任务(agent_type 作标题)
|    🔧 Grep: "agent_id"                           |
|    1m 02s · 子任务                               |
|  ------------------------------------------------ |
|  ● 等你处理  busyelf-app                      ×   |   ← 橙 waiting
|    需要授权运行 Bash 命令                         |   ← message(橙色,≤2 行)
|    2m 40s · claude-code                          |
|  ------------------------------------------------ |
|  ● docs(降存在感)                            ×   |   ← 绿点 done,标题/正文转灰
|    已完成:更新了 6 份文档…                        |   ← 最终回复摘要(≤2 行截断)
|    完成 · 3m 20s · claude-code                    |
|  ------------------------------------------------ |
|  ● proj-x   [rate_limit]                      ×   |   ← 红点 failed,红色 errorKind 小标
|    API Error: Rate limit reached                 |   ← 失败细节(红色,≤2 行)
|    失败 · 1m 02s · claude-code                    |
+--------------------------------------------------+
|  ⏹  全部结束 (4)                                 |   ← 仅有任务时出现
|  ☾ 保持屏幕唤醒                       [ off ]     |
|  ›  更多设置                                      |   ← 默认折叠(含 开机启动 / 网口 / 端口 / 超时)
|  ──────────────────────────────────────────────  |
|  Quit BusyElf                            ⌘Q      |
+--------------------------------------------------+
```

**任务行结构(四态共用三段竖排,≤2-3 行正文、尾部 `…` 截断;完整内容放 hover tooltip)**:

- **第 1 行(标题)**:状态点 + 标题(顶层=项目名 `cwd` basename;**子任务**=`↳` + agent 标签如 "Explore")+ failed 时的红色 `errorKind` 小标 + 右侧 `×`。
- **第 2 行(主信息,≤2 行)**:
  - `working`:当前动作——`🔧 tool: 细节` 优先,退化到最新回复(`MessageDisplay`),再退化"在干活…"。
  - `waiting`:`message`(橙色)。
  - `done`:最终回复摘要(`last_assistant_message`,灰色)。
  - `failed`:失败细节 / 错误原文(红色)。
- **第 3 行(元信息,1 行,灰字)**:`[可能已断 ·][完成/失败 ·] 时长 · agent [· 用户提示词]`。时长由 popover 打开时的 1s ticker 刷新(仅可见时运行,空闲 0 CPU);**终态时长冻结**在结束时刻。
- **状态点**:working/done = 绿 `●`,waiting = 橙,failed = 红;done 行整体**降存在感**(标题/正文转灰),failed 行醒目。working 长时间无进展加 `⚠` 暗示可能卡死(纯展示)。
- **疑似已断(看门狗)**:working 任务超过无活动阈值(默认 15min)→ 状态点**转灰**、`⚠` 显示、第 3 行前缀"**可能已断**"(表头同步显示"可能已断 · 已放行休眠")。表示已不再阻止休眠;状态仍是 working(不谎报完成/失败),**收到新进展会自动恢复**。
- **长文本**:换行/多空白先折叠成单逻辑行,再按 2 行截断——既不撑高也不撑宽。
- **Token**:`TaskSession`/协议已**预留**字段,但当前**不展示**(hooks 只暴露 subagent 级成本,不完整)。

### 终态生命周期(done/failed 的"看一次就消失")

终态项**不直接删**,留存展示,靠 **seen** 清理:

- **打开 popover**:把当前所有终态项标记 `seen` → 菜单栏角标(绿/红点)立即清除;本次仍显示它们。可见期间新变成终态的项也即时标 seen(绿点及时清,不残留)。
- **关闭 popover**:清理掉已 seen 的终态项 → **下次打开就不再显示**。
- **兜底**:超龄(默认 30 分钟)或终态项过多(上限 50)自动清理,避免用户从不打开导致堆积。

### 子任务(subagent)

`SubagentStart` 起一个独立子任务行,`session_id` 同父、靠 `agent_id` 区分(折进 task id)。展示上**整行缩进 + `↳ {agent 标签}`**,紧跟父行之下、与父行视觉粘连(不画分隔线)。父任务结束不级联结束子任务(各自收 `SubagentStop`);用户手动移除父任务则级联移除其子。

- **完成静默**:子任务完成**不点亮菜单栏绿点、不通知**(子任务是父任务的内部步骤、数量多,逐个通知是噪音;真正值得看的是父/顶层任务的完成)。子任务 working 仍照常阻止休眠、popover 仍展示其进展与结果。
- **已完成子任务列表封顶**:done 子任务保留上限 20(`TaskStore.maxDoneSubtaskCount`),超出按 `endedAt` 自动删最旧——防某个多子代理的父(如 workflow)完成后堆一长串。failed 子任务不在此静默删除(另有红点 / seen 生命周期,避免漏看失败)。

## 提醒(attention / failure)

- **working → waiting**(收到 `wait`):菜单栏进入橙色关注态;发系统横幅 `🔔 <项目>:<message>`,点击打开 popover。**只在进入 waiting 那一刻发一次**(去抖)。收到该任务 `update`(恢复 working)即清关注态。
- **→ failed**(收到 `StopFailure`/`fail`):菜单栏**红点**;发**紧急**横幅 `⚠️ <项目> 执行失败 [errorKind] 细节`。同样**只在跳变那一刻发一次**(去抖)。
- 横幅用任务 id 作通知标识 → 同任务重复只替换不堆叠。即使 Mac 已 idle 休眠,横幅会在唤醒后呈现,不影响"放行休眠"。需用户首次授权通知权限。

## 强制结束(force-stop)

用于"agent 崩了没发结束信号,任务残留阻止休眠",或主动清理某条。(残留任务即便不手动清,**看门狗也会在无活动超阈值后自动放行休眠**并标"可能已断";手动 `×` 用于立即清掉。)

- `×` 是安静的三级灰图标(`xmark.circle.fill`),hover 才变红,避免误点成主操作。
- 点击后,`×` **原地**换成两个标准小号按钮 —— `取消` 和红底的 `确认移除`(在 `×` 原位,**鼠标无需移动**即可点确认):

```
|  ● api-server                    取消  [确认移除]  |
|    🔧 Bash: pytest -x …                           |
|    12m 04s · claude-code · 修复登录超时…           |
```

- 全程在**同一行内**、行高几乎不变——不再有"切到另一处更高的确认视图被裁切"的问题。
- 若该任务**看起来仍活跃**(近期有 `update`),`确认移除` 加 `⚠` 前缀 + 警示 tooltip(`该任务似乎仍在活动…仍要移除?`),避免误移除导致 Mac 工作中途休眠。
- **只把任务从集合移除(解除其对休眠的阻止),不杀任何进程。** 移除父任务级联移除其子任务。
- 底部"全部结束 (N)"对所有任务套用同一流程(单段确认),仅有任务时出现。

## 底部设置 / overflow 菜单

- 常驻底部(唯一常用项):**"保持屏幕唤醒"**(额外持有 `PreventUserIdleDisplaySleep` 断言)。
- **"更多设置"** 折叠区(默认收起,点一下展开;每次重新打开 popover 自动收起,不抢视觉):
  - **开机启动**(`SMAppService.mainApp`,默认关)——不常用,收进折叠区省空间。
  - **监听所有网口 (0.0.0.0)** 开关 + 只读的实际监听地址(`监听 127.0.0.1:17872`,开了所有网口则 `0.0.0.0:…`)。切换即写 UserDefaults + 热重启监听(首次绑非回环触发系统本地网络授权)。
  - **端口** / **无响应超时(分钟)**:默认只读显示当前值 + 「修改」按钮,**点「修改」才就地变成输入框并聚焦**(避免输入框常驻聚焦抢眼);**回车应用、点别处放弃**。改端口会重启监听(需同步改 hook URL);改超时实时套用到看门狗(下限 1 分钟)。数值同样可用 `defaults write elf.busyelf {httpPort,inactivityTimeoutSeconds}`。
- 头部 `⋯` overflow 放低频项:版本号、About。
- 右键菜单镜像:打开面板 / 保持屏幕唤醒 / 开机启动 / 实际监听地址(只读)/ 监听所有网口 / 关于 / 退出(菜单做不了文本输入,端口/超时只在面板里改)。

## 视觉调性

呼应"elf"而不幼稚:一点点"小作坊"母题(空态的 ⚒、"工作台"字样)、暖色 accent、克制圆角,克制而非卡通。绿=完成、橙=等你、红=失败,语义直觉对齐。
