# BusyElf UI/UX 设计

> 术语对应 [PROTOCOL.md](PROTOCOL.md) 的状态:`working`(在干活,阻止休眠)、`waiting`(等用户,放行休眠但需关注)。

## 菜单栏图标

用单一 SF Symbol **`bolt.fill`(⚡︎)**,寓意"通电 / 保持清醒"。**只靠明暗、数字、着色变化传达状态,绝不替换字形**——换字形会改变宽度、让菜单栏抖动;且 `moon.zzz` / `zzz` / `powersleep` 等"睡眠"符号经核实**不在 SF Symbols 目录**,不可用。`bolt.fill` 是 template 图,自动适配深浅色菜单栏。

`button.imagePosition = .imageLeading`,图标在左、数字在右。

| 状态 | 外观 | 含义 |
|---|---|---|
| 无任务(N=0) | `[ ⚡︎ ]` 半透明(alpha≈0.45),**无数字** | 空闲,允许休眠 |
| 有 working | `[ ⚡︎ 2 ]` 全亮 + 数字 | 正在阻止休眠 |
| 有 waiting | `[ ⚡︎ 2 ]` 符号着 accent/橙色(+ 可选 `!`) | 有 agent 需要你处理 |

数字展示总任务数(working + waiting),钳制为 `9+`。**有意思的合理状态**:全部任务都 `waiting` 时,图标亮起"需要关注",但因无 `working`,**Mac 是允许休眠的**——agent 需要你,但没在空耗机器。

```
N=0 空闲:        [ ⚡︎ ]
2 个在干活:       [ ⚡︎ 2 ]
1 干活 1 等你:    [ ⚡︎ 2 ]  ← 符号着色提示"去处理"
```

交互:左键打开 popover;右键弹精简 `NSMenu`(Quit、About、开关)。

## Popover

`NSPopover` 锚定状态栏按钮,固定宽 ~320pt,内容用**纯 AppKit**(`NSViewController` + `NSStackView`/`NSView`,**不用** `NSHostingController`、不链接 SwiftUI——这是 ~12MB footprint 的关键,见 DESIGN.md §2)。结构:状态头 → 可滚动任务列表(超过 ~5 行滚动)→ 细分隔线 → 底部开关。

### 空态(N=0)

```
+--------------------------------------------------+
|  ⚡︎  BusyElf                                  ⋯  |
|  Idle · 允许休眠                                 |
+--------------------------------------------------+
|                                                  |
|                    ⚒                             |
|             工作台是空的。                        |
|        当前没有 agent 在工作。                    |
|     你的 Mac 会正常 idle 休眠。                   |
|     (注:合上盖子仍会休眠 — 长任务请开盖接电)     |
|                                                  |
+--------------------------------------------------+
|  ☾ 也保持屏幕唤醒                     [ off ]     |
|  ⏻ 开机启动                          [ off ]     |
|  ──────────────────────────────────────────────  |
|  Quit BusyElf                            ⌘Q      |
+--------------------------------------------------+
```

### 活跃态(1 working + 1 waiting)

```
+--------------------------------------------------+
|  ⚡︎  BusyElf                                  ⋯  |
|  Blocking sleep · 2 个任务                       |
+--------------------------------------------------+
|  ● 干活  api-server                          ×   |   ← 绿点 working
|         ▶ Bash: pytest -x                        |   ← 当前活动(来自 update)
|         12m 04s · claude-code                    |   ← 时长 · 来源
|  ------------------------------------------------ |
|  ⏸ 等你  busyelf-app                         ×   |   ← 橙 waiting,需要关注
|         需要授权运行 Bash 命令                    |   ← message(来自 wait)
|         2m 40s · claude-code                     |
+--------------------------------------------------+
|  ⏹  全部结束 (2)                                 |   ← 仅 N≥1 时出现
|  ☾ 也保持屏幕唤醒                     [ off ]     |
|  ⏻ 开机启动                          [ off ]     |
|  ──────────────────────────────────────────────  |
|  Quit BusyElf                            ⌘Q      |
+--------------------------------------------------+
```

**任务行结构**:
- 第 1 行:状态点 + 项目名(`cwd` basename,加粗)+ 右侧 `×`。
- 第 2 行:`working` 显示当前活动(`tool: detail`);`waiting` 高亮显示 `message`。
- 第 3 行:`已运行时长 · agent 来源` 灰字。时长由 popover 打开时的 1s ticker 刷新(仅可见时运行,空闲 0 CPU)。
- 状态点:`● 绿` = working;`⏸ 橙` = waiting;长时间未变化的 working 可加 `⚠` 暗示可能卡死(纯展示,不自动处理)。
- 长文本尾部截断,完整内容放 hover tooltip。

## 提醒(attention)

任务从 `working` → `waiting` 时(收到 `wait`):

- 菜单栏图标进入"需要关注"着色。
- 发一条系统横幅(`UNUserNotificationCenter`):`🔔 <项目>:需要你处理` + `message`。点击横幅打开 popover。
- **只在进入 waiting 那一刻发一次**(去抖,不刷屏)。
- 需用户首次授权通知权限。
- 收到该任务的 `update`(恢复 working)时,清除关注态。即使 Mac 已 idle 休眠,横幅会在唤醒后呈现,不影响"放行休眠"。

## 强制结束(force-stop)

用于"agent 崩了没发 `end`,任务残留阻止休眠"。

- `×` 是安静的三级灰色图标(`xmark.circle.fill`),hover 才变红,避免误点成主操作。
- 点击后该行**就地翻成单段行内确认**(不弹模态——模态会与 popover 失焦消失的生命周期打架):

```
|  ⏸ 等你  busyelf-app                             |
|   移除此任务,让 Mac 可以休眠?                    |
|   [ 移除 ]                                取消    |
```

- **只把任务从集合移除(解除其对休眠的阻止),不杀任何进程。**
- 若该任务**看起来仍活跃**(近期有 `update`),确认文案警示:`⚠ 该任务似乎仍在活动(12s 前还有进展)`,避免误移除导致 Mac 在工作中途休眠。
- 底部"全部结束 (N)"对所有任务套用同一流程,仅 N≥1 时出现。

## 底部 / overflow 菜单

- 常驻底部开关:**"也保持屏幕唤醒"**(额外持有 `PreventUserIdleDisplaySleep` 断言)、**"开机启动"**(`SMAppService.mainApp`,默认关)。
- 头部 `⋯` overflow 放低频项:About、检查更新、重置等。

## 视觉调性

呼应"elf"而不幼稚:一点点"小作坊"母题(空态的 ⚒、"工作台"字样)、暖色 accent、8pt 圆角,克制而非卡通。
